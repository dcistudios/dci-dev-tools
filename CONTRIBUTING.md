# Contributing to DCI Studios

First off, thank you for considering contributing to a DCI Studios public project! It’s people like you who help make our tools better for everyone.

By participating in this project, you agree to abide by our code of conduct and high-quality engineering standards.

## 🛠️ How Can I Contribute?

### Reporting Bugs
* **Check for existing issues:** Before opening a new one, please search the issue tracker to see if the bug has already been reported.
* **Use a clear title:** Summarize the issue in a few words.
* **Provide steps to reproduce:** Include as much detail as possible (OS, Browser/Node version, and specific input that caused the error).

### Suggesting Enhancements
We are always looking to expand our utility suite. If you have an idea for a new tool:
1. Open an issue with the tag `enhancement`.
2. Describe the utility and why it would be beneficial to the DCI ecosystem.

### Pull Requests (PRs)
1. **Fork the repo** and create your branch from `main`.
2. **Follow the style:** Match the existing vanilla JS/CSS architecture. We prioritize performance and zero-dependency solutions.
3. **Document your changes:** Update the `README.md` if you add a new tool.
4. **Self-Review:** Ensure your code is clean, commented, and free of console logs before submitting.

## 📜 Coding Standards

To maintain our "Architecting Excellence" philosophy:
* **Performance:** Ensure client-side operations are optimized for speed.
* **Security:** Never commit sensitive data, API keys, or internal DCI configurations.
* **Structure:** Keep `app.js` modular. New tools should be added to the `tools` object following the established pattern.
